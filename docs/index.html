Load direnv variables in GitHub Actions.

Install direnv, evaluate a trusted .envrc, export its environment variables to later workflow steps, and mask selected secret values.

Start with YAML View on GitHub
1Install the requested direnv release
2Run direnv allow and export json
3Validate required variables and mask secrets
4Expose variables to later workflow steps

Quick Start

Pin an exact release for predictable builds, or use @v1 to receive compatible updates within the current major version. 

steps:
  - uses: actions/checkout@v7
  - uses: HatsuneMiku3939/direnv-action@v1.3.6
    with:
      direnvVersion: 2.37.1
      masks: SECRET1, SECRET2

Inputs

Name Default Description
direnvVersion 2.37.1 The direnv version to install from GitHub release assets or cache.
masks '' Comma-separated environment variable names whose exported values should be masked in logs.
required '' Newline-delimited environment variable names that must be present after direnv export json.
path . Directory where direnv allow and direnv export json are executed.

Examples

Pin a direnv version

Use direnvVersion when workflows need a known direnv binary.

- uses: HatsuneMiku3939/direnv-action@v1.3.6
  with:
    direnvVersion: 2.37.1
Mask secret values

masks takes variable names, not raw secret values.

- uses: HatsuneMiku3939/direnv-action@v1.3.6
  with:
    masks: SECRET1, SECRET2
Require exported variables

Fail early when expected environment variables are missing.

- uses: HatsuneMiku3939/direnv-action@v1.3.6
  with:
    required: |
      AWS_REGION
      DATABASE_URL
      NODE_AUTH_TOKEN
Evaluate a subdirectory

Set path when the target .envrc is not at the repository root.

- uses: HatsuneMiku3939/direnv-action@v1.3.6
  with:
    path: child

Troubleshooting

Required variables are missing

Check that the selected path contains the expected .envrc and that direnv export json actually exports each name listed in required.

Secret values still appear in logs

Confirm that masks lists variable names such as SECRET1, not the secret values themselves.

PATH changes are not visible

When .envrc exports PATH, the action appends it through the GitHub Actions path API for later workflow steps in the same job.

Security Considerations

This action evaluates .envrc. Only run it with trusted repositories and trusted .envrc contents, especially when workflow secrets are available.

  • Review fork-based pull request workflows before running this action with secrets.
  • Treat masking as log redaction, not as a complete secret protection boundary.
  • Keep sensitive logic inside trusted workflow contexts whenever possible.
